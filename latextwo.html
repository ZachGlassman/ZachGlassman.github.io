How did we do it?

Let us take a look at the code you wrote for last section which had the code on the right. Here we want to look only at the first two lines and the last line. Notice that these are commands. In this section we will go over the basics of commands in general and learn how to setup a page so we can start writing. 

\documentclass{article}
\begin{document}
This is my first \LaTeX document!  
\begin{equation}
     a^2+b^2=c^2
\end{equation}
I can write equations!
\end{document}

Commands

The general syntax for commands in LaTeX is \command{argument}. In the example above, we have used four separate commands:

  • documentclass
  • begin
  • end
  • LaTeX

While the \LaTeX command is a nice touch and produces a fancily typeset LaTeX, the other three will be used in every document that you typeset! Lets take a closer look.






documentclass

The documentclass sets up what type of document you are working with, it loads many default styles and sets the overall look and feel of your document. There are also class specific commands. Some of the common classes and their uses are listed below. There are many more standard classes and even more custom classes. I usually use either the article class or the beamer class for pretty much everything unless there is a custom class required for homework preparation. It is important to remember that all of these classes are flexible and there is no right way to use them. However, every document must have class.

documentclass Usage
article For articles in scientific journals, presentations, short reports, program documentation, etc. Possibly the most versatile class and the one I recommend beginners use almost elusively at first.
report For longer report of several chapters, small book, thesis.
book For typesetting a book
memoir For books. Based on book class, but gives a bit more control over look and feel of document in a simpler manner.
letter For writing letters, has many predefined commands to help correct letter style.
beamer For writing presentations. As article class is to Microsoft Word, beamer class is to Microsoft Powerpoint.
begin and end

The begin and end tags signify the beginning and ending of an environment in the scope of your text. In the example above, they signify the end of the document environment, which is where the content to be shown is placed. We are going to explore the difference, try compiling two codes given below and see the difference.

\documentclass{article}
\begin{document}
\LaTeX~ is awesome
\end{document}

\documentclass{article}
\begin{document}
\begin{center}
\LaTeX~ is awesome
\end{center}
\end{document}

Note that these examples have the same input text, but different environments. Much of the control one has with LaTeX is by using such environments. I like to think of them as clothes, we can put certain input in multiple types and layers of clothes and the overall appearance will be determined by which clothes I use. However, just as clothing only changes appearance, but not substance, environments are simply ornamentation. This is why LaTeX is content driven, it allows the typesetter to focus on the content and not as much on how it appears..

Packages

Although with just the basic LaTeX package one could theoretically typeset any document, in practice this would be very difficult. Luckily, there are packages which extend functionality of the language in ways such as improved design elements, graphic generation, etc. The example code on the right uses many common packages. Notice that the usepackage command is invoked before the begin{document} command. Also note that some packages have option arguments which go in the square brackets before the curly brackets. 

\documentclass{article}
\usepackage[english]{babel}
\usepackage{amsfonts}
\usepackage{amssymb}
\usepackage{amsmath}
\usepackage{amsthm}
\usepackage{float}
\usepackage{graphicx}
\usepackage{enumerate}
\usepackage[margin=1in]{geometry}
\usepackage{tikz}
\begin{document}
I used some packages.
\end{document}

Below is a listing of some common packages and what they do. Many will be explained in more detail in subsequent sections.

Package Usage Documentation
amsmath Allows use of extra mathemathical symbols and adds features to facilitate writing mathematical formulas. amsmath documentation
geometry Set up page geometry such as margins. Takes arguments in brackets. geometry documentation
float Allows placement of figure and table environments to be fixed by user instead of by LaTeX float documentation
graphicx Allows image import of many different formats graphicx documentation
enumerate Allows lists to be enumerated by more than just number enumerate documentation
TikZ/PGF Vectorized drawing package producing high-quality images. TikZ/PGF documentation
subcaption Allows multiple in-line figures in a figure environment. Note: must be used with caption package. subcaption documentation

Often we have LaTeX commands that we use often and would rather not write out the entire command every time we need to use them. Occasionally, we are better off defining a macro which saves us a bit of work. The way in which this is done is by using the command newcommand and then defining our command. Below we have defined a command pder which produces a partial derivative. We then write out the full command in the first equation and use the macro in the second equation. Make sure that they come out the same! 

\documentclass{article}
\newcommand{\pder}[2]{\frac{\partial#1}{\partial#2}}
\begin{document}
\begin{equation}
\frac{\partial x}{\partial y}
\end{equation}
\begin{equation}
\pder{x}{y}
\end{equation}
\end{document}

In this case, our syntax is \newcommand{name}[# input arguments]{full command}. Note that in the full command part we reference our input arguments as #1,#2, etc. In general, macros speed the writing of the document, but slow the actual typesetting as the compiler must reference the definition. I tend to use them sparingly and usually for surprisingly simple tasks. Also remember that a macro is only useful if you are going to be using it over and over again.
